I modeled the bowl using the tablet-controlled turntable and my iPhone (13 mini). I put about 250 pictures into Reality Capture and it created two halves (inside and outside). I had to filter out the bag that the bowl sat on, and just generally clean up the two halves so they could come together cleanly. I used Mesh Lab to align and merge them.

I quickly realized that every software had trouble applying the texture map, so I had to go into blender and figure out how to fix the problem in UV map editing.

I just had to flip the mesh map around the x and y axis, and it applied correctly. I then exported it as a .glb, because it is a more compressed file (better for web rendering, especially because the color/texture is baked in).

I used Glitch to build a site with a model viewer component, so the user can interact with the model. I also added annotations to provide extra context (and make it fun).

And that's how it all happened! Thanks for taking a look!
